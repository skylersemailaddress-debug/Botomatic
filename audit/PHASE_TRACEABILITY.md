## Phase 1 Scope

Allowed in Phase 1:

- baseline gate execution
- evidence capture
- blocker classification
- workflow inventory and cleanup
- validator truth audit setup
- architecture/security/UX initial notes
- proof-tier correction needed to make baseline truthful

Not allowed in Phase 1:

- broad product redesign
- large feature builds
- commercial launch implementation
- max-power / 99% claim expansion
- UI rebuilds
- deployment-provider work beyond audit/gate classification

## Phase 1 Exit Criteria

Phase 1 may close only when:

- baseline workflow is green
- baseline evidence is recorded
- P0/P1 baseline blockers are classified or resolved
- proof tiers are defined
- workflow inventory is complete
- unnecessary workflow noise has a cleanup plan or PR
- initial architecture/security/UX notes exist
- Phase 2 entry criteria are documented
