# Universal Builder Target

Botomatic target: become a chat-first universal commercial app and website builder with 99 percent scenario coverage aspiration, while failing closed on unsafe or incomplete production paths.

Non-negotiable target contract:
- Chat is the primary control surface.
- No user-facing mode buttons or mode toggles.
- UI panels provide visibility into spec, assumptions, recommendations, contract readiness, and blockers.
- Build execution is blocked until a Build Contract is complete and approved.
- Generated app launch-readiness must be validator-proven.
- Placeholder/scaffold/fake implementation in production paths is launch-blocking.

High-risk areas that must not be silently auto-decided:
- payments and billing rules
- legal/compliance requirements
- auth/security model
- permissions and irreversible business actions
- data deletion/retention
- regulated workflows
- external API cost obligations
- public/private visibility rules

Current implementation truth:
- Control-plane foundation is production-grade for governed execution workflows.
- Spec intelligence, assumption governance, blueprint coverage, and generated-app validators are implemented as a universal-builder layer.
- Universal-builder readiness remains in progress until benchmark and validator evidence prove thresholds.
