# Botomatic

Botomatic is the next-generation Autobuilder control plane.

## MVP purpose

Build a chat-first system where:
- a conversational UI gathers app requirements
- the system compiles canonical master truth
- Autobuilder governs planning, support boundaries, proof, and promotion
- packetized execution drives implementation through external adapters
- GitHub Actions validates changes
- Vercel provides previews
- Supabase and Trigger.dev support backend and workflows

## Immediate build lane

This repo starts as the clean MVP control-plane scaffold.

### MVP active now
- master truth contract
- packet contract
- orchestrator API skeleton
- retained-spec classification
- legacy Autobuilder bridge plan

### Retained for post-MVP
- deeper proof/readiness integration
- advanced repair/replay
- expanded blueprint families
- richer autonomy scoring
- broader adapter depth

## Principles

- Autobuilder is the boss
- do not delete non-MVP specs
- contracts first, imports second
- autonomy must be bounded
- preserve evidence and readiness
